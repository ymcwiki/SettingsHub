# 无游戏环境怎么测试

没有安装《魔兽世界》也可以先拦住大部分逻辑错误和打包错误，但不能把自动测试写成“实机已验证”。

## 现在能自动检查的内容

- 设置枚举、搜索、写入、撤销、重放、快照和推荐包等核心逻辑
- 中英文案、策展数据、标签和关键字段是否完整
- 缺少部分新旧版本 WoW API 时，插件是否安全降级
- Lua 文件能否在模拟的 WoW 环境里加载
- Windows 与 Linux 是否得到相同的测试结果
- 发布压缩包的目录、版本号以及是否误带开发文件

## 现在不能替代实机检查的内容

- 窗口布局、字号、滚动、鼠标点击和右键菜单是否顺手
- 战斗中受保护设置是否真的被客户端拦截，退出战斗后是否正确执行
- 是否触发 taint、blocked action 或其他只在游戏客户端出现的问题
- 重登、换角色、服务器同步和其他插件覆盖设置的真实表现
- 聊天窗口、手柄、声音、编辑模式等系统的最终交互效果

这些项目继续保留在 [ACCEPTANCE.md](ACCEPTANCE.md) 和 [VERIFIED.md](VERIFIED.md)，等有游戏环境时逐项打勾。

## 本地跑法

首次准备：

```powershell
python -m pip install -r requirements-dev.txt
```

运行全部无头测试：

```powershell
python tests/run_headless.py
```

检查发布包：

```powershell
python scripts/package.py smoke
```

提交或合并后，GitHub 还会在 Windows 和 Linux 各跑一遍。只有这些检查通过，打版本标签后的发布任务才会继续。

## 当前发布口径

自动测试通过时，可以说“核心逻辑与打包检查通过”。在实机清单完成前，不说“完整兼容”或“全部功能已验证”。
