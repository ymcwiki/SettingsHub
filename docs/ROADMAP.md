# 路线图

2026-07-17 定稿。原则:v0.1.x 只修不加;新功能守 SPEC 第 6 节非目标(不做 QoL 大杂烩、不做通用框架、不静默改值);策展扩容是每版本例行,不是一次性项目。

## v0.1.x 地基(进行中,只修不加)

- 实机验收(docs/ACCEPTANCE.md 逐项),UI 磨合报错修复
- VERIFIED.md 八项 UNVERIFIED 回写,解锁 verify 隐藏的策展项
- 官方设置搜索命中数实测调优(F2.4 的 ≥20 项口径)
- 出口条件:验收清单全勾,taintLog 干净,才开 v0.2 分支

## v0.2 国际化与备份(代码完成于 2026-07-17,发布 v0.2.0;实机验收见 ACCEPTANCE.md v0.2 节)

实现偏差说明:快照没有挤进日志页,独立成「快照」页(列表 + 对比基准切换 + 差异勾选恢复需要整页空间);冲突区按计划放在日志页。

**T1 enUS 本地化**
- 数据模型调整:`text = { zh, en, keywords }`(现 `text.en` 是关键词数组,改名 `keywords`,`en` 留给英文白话说明),census2lua 与 Search/Widgets/Integration 同步迁移
- locale 机制:`ns.L` 字符串表,GetLocale 判定,enUS 为默认,zhCN/zhTW 走中文表
- UI chrome 全部硬编码中文抽表(约百条)
- 106 条策展文案逐条写英文说明(人话,禁机翻直出;这是护城河的翻译,预算照策展标准)
- 验收:enUS 客户端全 UI 无中文残留;中英俗名搜索都命中;无头测试加断言(每条 curated 的 zh/en/keywords 三字段齐全);CurseForge 描述页移除「UI 目前是简中」的限制声明

**T2 命名快照与恢复**
- 快照 = 全量枚举值(不只本插件改过的)落 SV:名字 + 时间戳 + build 号,上限 10 份,超限先删最旧(带确认)
- 快照列表 UI(日志页加「快照」区),操作:新建、删除、两份对比(复用 dump_diff 的五类口径:新增/删除/值变/scope/secure)、选择性恢复
- 恢复逐条走写管线(source=snapshot),全程可撤销
- 验收:实机「快照、改乱二十项、对比、恢复」回环一致;补丁场景演练(对比两 build 快照能标出版本差异);无头测试覆盖回环与上限淘汰

**T3 冲突检测面板**
- 聚合 blame 与 replay 补写记录:同一 key 被同一外部来源跨登录覆盖 ≥3 次判为冲突
- 日志页加「冲突」区:列插件名、涉及项、次数;每条给两个处置:停止管理该项(移出期望态)或保持(继续每次改回)
- 验收:用一个最小测试插件模拟持续覆盖,冲突条目出现且两种处置行为正确

## v0.3 策展扩容与场景包(代码完成于 2026-07-17,发布 v0.3.0;实机验收见 ACCEPTANCE.md v0.3 节)

- **场景化推荐包**:已实现(新「推荐包」页)。「晕 3D 舒适」「团本性能」「PvP 信息」「新手推荐」四个包,每包一组 curated 值(测试强制只引用策展项),预览确认后应用,LogBulk 整包可撤销
- **策展 +39 条**:已实现。新主题 I 聊天(8)/ J 手柄与光标(15)/ K 任务与地图(12),B 主题补软目标 tooltip(3);存疑项按惯例标 verify(VERIFIED.md 第 10/11 条)
- **聊天窗口域**:已实现(Adapters/ChatWindow.lua,第八个批量域),纳入 profile 与导入导出;UI 即时刷新语义待实机验证(VERIFIED.md 第 9 条)
- **拼音搜索**:已实现。gen_pinyin.py 从策展文案用字生成字表(736 字,13KB,预算 100KB),搜索索引加全拼与首字母(xingmingban / xmb 命中姓名板)

## v0.4 从工具到向导(代码完成于 2026-07-18,发布 v0.4.0;实机验收见 ACCEPTANCE.md v0.4 节)

用户反馈「功能太基础」后回知识库规划的一轮:调研第二章明确「纯浏览器天花板低」「策展式推荐引导层是空白」,v0.4 就是把引导层做出来。

- **发现页**(新默认首页):声明式主动建议(Data/Guides.lua 的 tips,条件命中才显示,可一键处理/不再提示)+ 八条意图引导(items 引用策展 id,内联真控件)+ 补丁新增一览
- **试穿模式**(Core/Trial.lua):临时应用 source=trial 不进期望态,到期/登出/手动还原,可转常驻;崩溃残留登录兜底
- **手感**:范围数值真滑块(松手提交防日志刷屏)、枚举现代下拉(带循环按钮回退)、行悬停高亮、手搓小地图按钮
- **官方新系统配套**:12.0.7 差集池只有 5 条可产品化(辅助作战/冷却管理器),4 条并入 E 主题,独立主题等 12.1 dump 扩池后再立

## 结构性风险

- 单人维护断档是 AIO 的死因。对冲:L1 管线已自动化、数据格式低门槛、文档齐;但策展扩容需要「每版本固定半天」的持续投入,断两个补丁元数据变负资产(调研报告第九章结论)
- 13.0 大版本预留 1 到 2 天集中适配,PTR 上提前跑 dump
