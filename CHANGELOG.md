# Changelog

## v0.5.3(2026-07-19)

浏览器左侧分类改版:主题分组取代无意义的数字类别。

- **19 个中文主题分组**(相机与视角、姓名板、图形与画质、声音与语音、动作条与技能、任务与地图……):此前左栏是控制台的数字类别,其中「Game」一个桶就装了一千多条等于没分类。现在按 CVar 名字家族运行时归类,每条落一个主题,侧栏与其他分类同一套 UI(计数、选中高亮、点击过滤)
- 约 76% 的 CVar 落进具体主题,其余归「其他」(名字无家族规律的引擎内部项);分类规则在 Data/Topics.lua,改一处顺序即可微调
- 主题文本也进搜索:搜主题名或家族词能定位

## v0.5.2(2026-07-18)

系统性代码审查(五维)后的修复,三批共 8 处正确性/安全/数据完整性缺陷。详见 docs/REVIEW-2026-07-18.md。

- **数据不再丢失**:全量还原/回默认只对写成功的键清恢复依据,失败或战斗排队的键保留原值可重试(此前无条件清空全表);试穿还原逐项成功才删恢复记录;整包撤销不再把旧值写进当前 profile 的期望态改坏定义
- **诊断与自测彻底只读**:`/sh diag` 可写性扫描改读只读标志、不再同值写回;`/sh probe` 去掉程序化点击;`/sh test` 实机只读,写管线测试搬到 CI。落实「诊断绝不改配置」
- **导入串安全收口**:分享串加解压上限、拒尾随字节、递归结构校验;console 命令白名单(只认策展登记的命令,挡住损坏存档注入任意命令);未知/只读 CVar 预览与应用一致真跳过
- **战斗一致性**:内部写异常不再让归因追踪停摆;脱战补写诚实区分成功失败;战斗中撤销/回默认排队后正确提交状态;战斗中含键位/宏/聊天窗口等域的 profile 切换整体拒绝而非半切换

## v0.4.3(2026-07-18)

- **修复:注册进官方设置搜索的 28 个策展项改不动**(实机验收首日报告的「勾选框点不动」悬案闭环)。根因:插件写入成功后通知官方设置面板刷新,官方 setting 却从注册时缓存的旧值表读值,又经注册的回调把旧值写回,一来一回用户的修改被当场抵消。受影响的不止勾选框:推荐包应用、登录重放、快照恢复对这批键的写入同样被静默抵消。现在通知前先同步值表、通知期间屏蔽回调
- 无头测试补上官方 Settings API 桩(仿实机回调语义),Integration 从「只查语法」升级为真实执行,UI 桩加 no-op 兜底;断言总数 161 项。此前有 3 处既有断言的失败被「语法检查不执行」掩盖,本次一并暴露并由同一修复解决

## v0.4.1(2026-07-18)

内部错误自检 + 诊断只读化(「一定不能乱改配置」约束落地)。

- **错误自动捕获**:事件分发、斜杠命令、全部控件点击/提交回调、页面监听器统一包上错误捕获;出错记入会话表与 SavedVariables 环(保留 20 条,跨 /reload),聊天框每种错误提示一行,`/sh diag` 显示最近错误与调用栈首行。不再依赖 scriptErrors 弹窗截图
- **单点错误不再连坐**:某个页面回调出错不会阻断写入本身,其他页面照常刷新
- **`/sh diag` 彻底只读**:去掉写入回环测试(此前会写 cameraZoomSpeed 再撤销),换成 db/撤销环/读值/引擎的只读体检;可写性扫描保持「写回当前原值」零改动语义,测试断言 diag 前后撤销环与值分毫不动
- **`/sh test` 不再自动布置跨重登标记**:第三组(会临时改一项服务器存储值直到重登)改为显式 `/sh test relog`;已有残留标记时任何 test 都会完成收尾并复原
- 无头测试增至 149 项断言
